    属性区

    属性区窗口显示在引擎的右侧，使用属性区窗口可以查看和编辑BCreator.io编辑器中几乎所有内容的属性和设置。

    打开属性区窗口

    请点击自定义预制件按钮，以打开属性区窗口。

    控制属性区窗口焦点

    默认情况下，属性区窗口显示当前选择的属性。

    检查项目

    在属性区窗口中可以看到和编辑的内容取决于您选择的内容。本节描述了属性区窗口针对您可以选择的不同类型的项目显示的内容。

    检查游戏对象

    选择游戏对象时（例如，在资源管理或场景视图中），属性区窗口将显示其所有组件和材料的属性。您可以编辑属性，然后在属性区窗口中重新排列组件。

    检查资源

    从资源管理窗口中选择资源时，属性区显示的设置可控制BCreator.io在运行时如何导入和使用资源。

    检查预制件

    使用预制件时，属性区窗口将显示一些其他信息并提供一些其他选项。

    属性面板

    属性面板用于展示当前资源的所有属性组件，并可以右键点击组件对其进行的操作。

    • 移除组件：删除该组件
    • 上移：将组件位置上移
    • 下移：将组件位置下移
    • 复制组件：复制该组件
    • 粘贴新组件：粘贴复制的组件
    • 粘贴组件值：将复制组件的数值粘贴到该组件（只针对相同组件）

    静态

    如果是游戏对象中不会在运行时移动，它称为静态游戏对象。如果游戏对象在运行时移动，则称为动态游戏对象。

    BCreator.io中的许多系统都可以在编辑器中预先计算有关静态游戏对象的信息。由于游戏对象不移动，因此这些计算的结果在运行时仍然有效。这意味着BCreator.io可以节省运行时的计算量，并有可能提高性能。

    静态编辑器下拉菜单在属性区中查看游戏对象时可见。

    在静态编辑标记属性列举了一些BCreator.io系统中可预先计算的静态游戏物体。使用下拉列表定义哪些系统应在其预计算中。在运行时设置静态编辑器标志对这些系统没有影响。

    对于需要了解该游戏对象的系统，只应在预计算中包括游戏对象。对于不需要了解游戏对象的系统，在预计算中包含游戏对象可能会导致计算浪费，不必要的大数据文件或意外行为。

    在静态编辑标记属性位于属性区的右上角。它包括一个下拉菜单，可让您选择想要设置的值。

    属性 功能
    非静态 对于任何系统，请勿在预计算中包含GameObject。
    静态光影 标记对象以生成光照图。提前对一个场景中的对象光照进行计算。
    完全静态 在以下所有系统的预计算中都包含GameObject。

    组件分类

    模型、渲染、物理、音频、特效、事件、UI界面、导航网格、脚本、后期渲染。

    空间变换（组件）

    添加预制件默认添加空间变换（组件），位置XYZ坐标表示预制件位于地图中的坐标点，旋转的XYZ表示预制件围绕XYZ轴的旋转角度，缩放的XYZ表示在预制件XYZ方向上的比例。

    模型

    包括模型（组件）和动态模型。

    在模型分类中选择模型的类型进行添加，添加后可选择模型资源文件添加到组件，设置模型颜色、材质，动态模型可添加骨骼文件。

    物理

    早期的游戏并没有强调物理引擎的应用，当时无论是哪一种游戏，都是用极为简单的计算方式做出相应的运算就算完成物理表现，如超级玛丽和音速小子等游戏。

    较为常见的物理处理是在跳跃之后再次落到地上，并没有特别注重物理表现效果。

    当游戏进入三维时代后，物理表现效果的技术演变开始加速，三维呈现方式拓宽了游戏的种类与可能性，越来越好的物理表现效果需求在短时间内大幅提升。

    如何制作出逼真的物理互动效果，而又不需要花费大量时间去撰写物理公式，是物理引擎重点要解决的问题。

    在BCreator.io中，物理引擎是游戏设计中最为重要的步骤，主要包含刚体、碰撞、物理材质以及关节运动等。

    游戏中物理引擎的作用是模拟当有外力作用到对象上时对象间的相互影响，比如赛车游戏中，驾驶员驾驶赛车和墙体发生碰撞，进而出现被反弹的效果。

    物理引擎在这里用来模拟真实的碰撞后效果。通过物理引擎，实现这些物体之间相互影响的效果是相当简单的。

    包括刚体、盒型碰撞体、球形碰撞体、胶囊碰撞体、模型碰撞体、车轮碰撞体、角色控制器、固定关节、弹簧关节、铰链关节、角色关节、可配置关节、恒力，该类组件的详情将在手册中的物理部分进行更加详细的介绍。
